# 型
Haxeコンパイラは、コンパイル時にプログラム内の型に関連するエラーを検出するのに役立つ高級型システムを採用しています。<br>
型エラーとは、文字列で割ること、整数のフィールドにアクセスしようとする、引数が少なすぎる関数を呼び出すなど、与えられた型に対して無効な操作を行うことを指します。

一部の言語では、この追加の安全性には代償が伴います。なぜなら、プログラマーは構文構造に明示的に型を割り当てざるを得ないからです:
``` as3
var myButton:MySpecialButton = new MySpecialButton(); // As3
```
``` c++
MySpecialButton* myButton = new MySpecialButton(); // C++ 
```
Haxeでは明示的な型注釈は必要ありません。なぜなら、コンパイラが次の型を推論できるからです:
``` haxe
var myButton = new MySpecialButton(); // Haxe
```
型理論については、後の「型理論」のセクションで詳しく解説します。(2026/07/12現在、非記載)<br>
現時点では、上記のコードにある変数 `myButton` が **クラス `MySpecialButton` のインスタンス** であることがわかっている、とだけ述べておけば十分でしょう。

Haxeの型システムでは、7つの型グループが定義されています↓

- `Class`インスタンス: 特定のクラスまたはインターフェースのオブジェクト
- `Enum`インスタンス: Haxeの列挙型の値
- `Structure`: 匿名構造体、すなわち名前付きフィールドの集合
- `Function`: 複数の引数と1つの戻り値を持つ複合型
- `Dynamic`: 他のあらゆる型と互換性のあるワイルドカード型
- `Abstract`：コンパイル時に定義され、実行時には別の型で表現される型
- `Monomorph`: 後で別の型になる可能性のある未知の型

これらのタイプ群のそれぞれについて、またそれらが互いにどのように関連しているかについては、以降の章で説明していきます。
