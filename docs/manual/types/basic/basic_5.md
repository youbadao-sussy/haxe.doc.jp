# Nullability
!!! info "Define: nullable"
    _Haxe において、ある型は、`null` がその型の有効な値である場合に、**null許容型**とみなされます。_

プログラミング言語では、NULL許容性について単一で明確な定義が設けられていることが一般的です。<br>
しかし、Haxeはターゲット言語の性質上、この点において妥協点を模索せざるを得ません。<br>
ターゲット言語の中には、あらゆる値に対して`null`を許可し、実際、デフォルト値として`null`を採用しているものもあれば、特定の型に対しては`null`を一切許可していないものもあるからです。<br>
このため、ターゲット言語を以下の2種類に区別する必要があります。

!!! info "Define: 静的ターゲット"
    _静的ターゲットでは、独自の型システムが採用されており、基本型において `null` は有効な値とはみなされません。_<br>
    _これは、Flash、C++、Java、および C# の各ターゲットに当てはまります。_
!!! info "Define: 動的ターゲット"
    _動的ターゲットは型に対する制約が緩く、基本型に対して `null` 値を許可しています。_<br>
    _これは、JavaScript、PHP、Neko、および Flash 6～8 のターゲットに適用されます。_

動的ターゲットで `null` を扱う場合、特に心配する必要はありませんが、静的ターゲットの場合は少し注意が必要になる場合があります。<br>
まず、基本型はデフォルト値で初期化されます。

!!! info "Define: デフォルト値"
    _基本型は、静的ターゲットにおいて以下のデフォルト値を持ちます。_
    - `Int`: `0`
    - `Float`: Flash では `NaN`、その他の静的ターゲットでは `0.0`
    - `Bool`: `false`

その結果、Haxe コンパイラでは、静的ターゲットにおいて基本型に `null` を代入することは許可されていません。<br>
これを実現するには、基本型を `Null<T>` でラップする必要があります。
``` haxe
var a:Int = null; // 静的プラットフォームではエラー
var b:Null<Int> = null; // 有効
```

同様に、基本型はラップされていない限り、`null` と比較することはできません。
``` haxe
var a:Int = 0;
if( a == null ) { ... } // 静的プラットフォームではエラー

var b:Null<Int> = 0;
if( b != null ) { ... } // 有効
```
この制限は、統一が行われるあらゆる状況に適用されます。

!!! info "Define: `Null<T>`"
    _静的ターゲットでは、`Null<Int>`、`Null<Float>`、`Null<Bool>` といった型を使用することで、`null` を値として許可することができます。_<br>
    _動的ターゲットでは、これによる効果はありません。_<br>
    _また、`Null<T>` を他の型と組み合わせて使用することで、`null` が許容される値であることを明示することもできます。_

`Null<T>` や `Dynamic` に `null` 値が「隠されている」状態で、それが基本型に代入された場合、デフォルト値が使用されます。

``` haxe
var n : Null<Int> = null;
var a : Int = n;
trace(a); // 静的プラットフォームでは0
```
