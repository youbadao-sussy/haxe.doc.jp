# オーバーフロー
パフォーマンス上の理由から、Haxeコンパイラはオーバーフロー時の挙動を一切強制しません。<br>
オーバーフローのチェックは、ターゲットプラットフォーム側の責任となります。<br>
オーバーフローの挙動に関するプラットフォームごとの注意事項↓

- C++、Java、C#、Neko、Flash: 32ビットの符号付き整数で、通常のオーバーフロー処理が適用される。
- PHP、JS、Flash 8: ネイティブな整数型（Int）が存在せず、数値が浮動小数点の上限（252）に達すると精度が失われる。

あるいは、**haxe.Int32** および **haxe.Int64** クラスを使用することで、特定のプラットフォームでは余分な計算が必要になるものの、オーバーフロー時の挙動を正しく保証することができます。
