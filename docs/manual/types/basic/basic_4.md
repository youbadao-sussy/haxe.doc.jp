# Void
!!! info "Define: Void"
    _型の存在しないことを示す。何か（通常は関数）に値がないことを表現するために用いられる。_

`Void` は、実際には型ではないため、型システムにおける特殊なケースです。<br>
これは型の不在を表すために使用され、主に関数の引数や戻り値の型に適用されます。<br>
最初の「Hello World」の例で、すでに `Void` を「目にした」はず。
もう一度見てみましょう。
``` haxe
/**
 *  ドキュメント用の複数行コメント。
 */
class Main {
    static public function main():Void {
        // 単一行コメント
        trace("Hello World");
    }
}
```

関数の型については、「関数の型」のセクションで詳しく解説しますが、ここで簡単に触れておきます。<br>
上記の例における関数 `main` の型は `Void->Void` であり、これは「引数がなく、何も返さない」という意味です。<br>
Haxe では `Void` 型のフィールドや変数は許可されておらず、そのような宣言を行うとエラーになります。<br>
例:
``` haxe
// Void 型の引数および変数は使用できません。
var x:Void;　// これだとエラーになる。
```
