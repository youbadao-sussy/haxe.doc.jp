# 沿革
Haxeプロジェクトは、フランスの開発者**Nicolas Cannasse**によって2005年10月22日に開始されました。<br>
これは、人気のオープンソースであるActionScript 2コンパイラである**MTASC**(Motion-Twin Action Script Compiler)および社内言語**MTypes**の後継であり、型推論をオブジェクト指向言語に応用する実験を行っていました。<br>
ニコラスの長年のプログラミング言語設計への情熱と、**Motion-Twin**でのゲーム開発者としての仕事の一環として異なる技術を融合させる新たな機会の出現が、まったく新しい言語の創出へとつながりました。

当時は**haXe**と綴られていたベータ版は2006年2月にリリースされ、<br>
最初のサポート対象はAVMバイトコードとニコラス自身の**Neko**仮想マシンでした。

現在もHaxeプロジェクトのリーダーであるNicolas Cannasseは、明確なビジョンを持ってHaxeの開発を続け、2006年5月にHaxe 1.0がリリースされました。この最初の大規模リリースにはJavaScriptコード生成のサポートが含まれ、今日のHaxeを定義するタイプ推論や構造的サブタイピングなどの機能がすでに備わっていました。

Haxe 1は2年間でいくつかの小規模なリリースがあり、2006年8月にはFlash AVM2ターゲットと**haxelib**-tool、2007年3月にはActionScript 3ターゲットが追加されました。この期間中、安定性の向上に強く注力し、いくつかの小規模なバグ修正がリリースされました。

Haxe 2.0は2008年7月にリリースされ、**Franco Ponticelli**の提供によるPHPターゲットが含まれていました。**Hugh Sanderson**による同様の取り組みにより、2009年7月のHaxe 2.04リリースでC++ターゲットが追加されました。

Haxe 1と同様に、その後数か月にわたる安定性のリリースが続きました。2011年1月、Haxe 2.07が**マクロ対応**の形でリリースされました。<br>
その頃、**Bruno Garcia**がJavaScriptターゲットのメンテナとしてチームに加わり、その後の2.08および2.09リリースで大幅な改善が見られました。

2.09のリリース後、**Simon Krajewski**がチームに加わり、Haxe 3の開発が始まりました。<br>
さらに、**Cauê Waneck**のJavaおよびC#ターゲットはHaxeビルドにも取り入れられました。<br>
その後、最後のHaxe 2リリースが決定され、2012年7月にHaxe 2.10がリリースされました。

2012年末、Haxe 3のスイッチが切り替わり、新たに設立された**Haxe Foundation**の支援を受けたHaxe Compilerチームは、この次の大規模バージョンに注力しました。<br>
その後、Haxe 3は2013年5月にリリースされました。
